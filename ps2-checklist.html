Problem Space 02 — Checklist

HCP Engagement & Prioritization

⏱ Total Time: ~45 minutes 🎯 6 Steps 🏆 100 Points

Test your agent directly to validate it meets the judging criteria. Each step contains prompts to paste directly into your agent and checklists to track your score.

📋 Process Overview

Each step contains prompts to paste directly into your agent. The goal is to test whether your agent responds accurately, intelligently, and in line with the problem it was built to solve. Complete the checklists after each response to track your score.

Step 1
Business Impact Assessment
⏱ 10 minutes  ·  25 points
🎯 PurposeTest whether your agent understands the business problem and can articulate value.
Prompt 1A — Problem Awareness Check
Paste this prompt into your agent
Which HCPs in my territory have not yet given digital consent under CASL
and why should I prioritize them over others?
Prompt 1B — ROI Articulation
Paste this prompt into your agent
How much time and resource am I currently wasting by reaching out to
low-value non-consented HCPs? What would change if I focused only on
high-priority targets?
Prompt 1C — User Relevance
Paste this prompt into your agent
I'm an MSL managing a portfolio of HCPs across Ontario. Which ones
should I be visiting in person this month to maximize consent success?
What to Look For in the Response
  • Does the agent identify non-consented HCPs accurately?
  • Does it connect prioritization to business outcomes (ROI, consent rates)?
  • Does it tailor recommendations to the user’s role (rep vs. MSL)?
  • Does it reference CASL constraints and the “one meaningful chance” context?
  • Does it provide actionable, specific guidance rather than generic responses?
  • Red flag: Generic consent advice without CASL-specific context

✅ Checklist — Business Value (25 points)

Step 2
Prioritization Intelligence Validation
⏱ 12 minutes  ·  20 points
🧠 PurposeTest whether your agent’s scoring model is accurate, explainable, and actionable.
Prompt 2A — Scoring Logic
Paste this prompt into your agent
Show me the top 10 non-consented HCPs in my region ranked by priority.
Why is each one ranked where they are?
What to Look For in the Response
  • Does the agent use both CRM engagement data AND IQVIA prescriber data in scoring?
  • Can it explain WHY an HCP is ranked HIGH/MED/LOW?
  • Are the rankings defensible and consistent with the data?
  • Does it provide the top 10 in a clear, actionable format?
  • Red flag: Rankings without explanation of scoring logic
Prompt 2B — Data Integration Check
Paste this prompt into your agent
Dr. Sarah Chen is a rheumatologist in Vancouver. She's in the top decile
for prescriptions but has had no rep visits in the last 6 months.
What priority tier is she and what consent strategy do you recommend?
What to Look For in the Response
  • Does the agent factor in both high prescription volume and low engagement?
  • Does it assign a clear priority tier (HIGH given the profile)?
  • Does it recommend a specific, personalized consent approach?
  • Does it explain the reasoning behind the tier assignment?
  • Red flag: Generic response that doesn’t integrate the two data signals
Prompt 2C — Tier Strategy Mapping
Paste this prompt into your agent
Give me the recommended consent approach for a HIGH priority HCP
versus a LOW priority HCP. How should my outreach differ?
What to Look For in the Response
  • Does the agent recommend differentiated strategies per tier?
  • Does it distinguish in-person vs. digital approaches by tier?
  • Does it factor in resource allocation (time, effort)?
  • Is the advice practical and actionable for field teams?
  • Red flag: Same advice for all tiers — no differentiation
Prompt 2D — Edge Case Handling
Paste this prompt into your agent
I have an HCP who was previously contacted for consent and declined.
How does your prioritization handle this? Should I try again?
What to Look For in the Response
  • Does the agent handle the declined consent scenario intelligently?
  • Does it reference CASL’s “one meaningful chance” constraint?
  • Does it recommend a nuanced approach (not just “try again”)?
  • Does it deprioritize declined HCPs appropriately?
  • Red flag: Ignores the decline history or recommends re-contacting inappropriately

✅ Checklist — Prioritization Intelligence (20 points)

Step 3
Scalability & Regulatory Adaptability
⏱ 8 minutes  ·  20 points
🌍 PurposeTest whether your agent can operate across markets and regulatory frameworks.
Prompt 3A — Multi-Market Awareness
Paste this prompt into your agent
I'm expanding this consent prioritization approach to our team in France.
What changes under GDPR compared to CASL? How does the prioritization
strategy need to adapt?
What to Look For in the Response
  • Does the agent demonstrate awareness of CASL vs. GDPR differences?
  • Does it identify specific prioritization logic changes required under GDPR?
  • Does it address data usage constraints in each jurisdiction?
  • Does it position the solution as adaptable across markets?
  • Red flag: Treats all jurisdictions the same with no regulatory nuance
Prompt 3B — Cross-Therapeutic Flexibility
Paste this prompt into your agent
This tool was built for our immunology team. Can it work for our
oncology field force? What would need to change in the prioritization
model for a different therapeutic area?
What to Look For in the Response
  • Does the agent identify differences between therapeutic area needs?
  • Does it explain what scoring factors would change for oncology?
  • Does it demonstrate flexibility in the underlying model?
  • Is it honest about what would require reconfiguration?
  • Red flag: Claims the model works the same for all therapeutic areas
Prompt 3C — Data Source Adaptability
Paste this prompt into your agent
In Australia, we don't have access to IQVIA data. What alternative
data sources could be used to maintain prioritization quality?
What to Look For in the Response
  • Does the agent identify credible alternative data sources?
  • Does it explain how prioritization quality would be maintained?
  • Does it acknowledge honest trade-offs from losing IQVIA data?
  • Does it show flexibility in the data architecture?
  • Red flag: Assumes IQVIA or equivalent data will always be available

✅ Checklist — Scalability & Reach (20 points)

Step 4
Production Readiness & Data Integrity
⏱ 8 minutes  ·  15 points
⚙️ PurposeTest whether your agent handles real-world data quality and operational challenges.
Prompt 4A — Data Freshness
Paste this prompt into your agent
When was the CRM and IQVIA data last updated that you're using
for prioritization? How does stale data affect my recommendations?
What to Look For in the Response
  • Does the agent acknowledge data freshness and its impact?
  • Does it specify update cadence for CRM and IQVIA data?
  • Does it explain how stale data degrades prioritization quality?
  • Does it communicate data limitations transparently?
  • Red flag: No mention of data freshness or when data was last updated
Prompt 4B — Missing Data Handling
Paste this prompt into your agent
I have 50 HCPs in my territory with no CRM visit history and no
IQVIA prescriber data. How do you prioritize them?
What to Look For in the Response
  • Does the agent handle zero-data HCPs with a fallback strategy?
  • Does it explain what signals are used when primary data is absent?
  • Does it assign a default tier with clear reasoning?
  • Does it flag these HCPs for manual review or additional data collection?
  • Red flag: Excludes or ignores HCPs with missing data
Prompt 4C — Compliance Guardrails
Paste this prompt into your agent
Can you show me the consent status and outreach history for
Dr. James Park? What data are you using and how is it governed?
What to Look For in the Response
  • Does the agent provide an audit trail for consent and outreach history?
  • Does it explain what data sources are used for this HCP?
  • Does it demonstrate compliance and data governance awareness?
  • Does it explain how data access is controlled?
  • Red flag: No mention of compliance or data governance
Prompt 4D — Workflow Integration
Paste this prompt into your agent
How does this tool fit into my daily workflow in Veeva CRM?
Can I action these recommendations directly from the prioritization list?
What to Look For in the Response
  • Does the agent describe integration with Veeva CRM or equivalent tools?
  • Does it explain how recommendations are actioned in existing workflows?
  • Does it minimize context switching for the field team?
  • Does it address realistic adoption barriers?
  • Red flag: No mention of Veeva or real field tool integration

✅ Checklist — Production Readiness (15 points)

Step 5
Innovation & Technical Excellence
⏱ 5 minutes  ·  15 points
🛠️ PurposeTest the depth, adaptability, and intelligence of your agent.
Prompt 5A — Adaptive Learning
Paste this prompt into your agent
Over the last quarter, my HIGH priority HCPs had a 40% consent success
rate but my MED priority HCPs had a 60% success rate. What does this
tell you and how should the scoring model be adjusted?
What to Look For in the Response
  • Does the agent recognize the scoring model inversion?
  • Does it identify potential causes (e.g., over-contacted high-prescribers)?
  • Does it suggest specific model recalibration?
  • Does it propose a feedback loop for continuous improvement?
  • Red flag: Static response that doesn’t adapt to outcome data
Prompt 5B — Conversational Intelligence
Paste this prompt into your agent
I have 3 hours free next Tuesday. Build me an optimized visit schedule
for consent outreach in the Toronto downtown core, prioritizing
HIGH tier HCPs first.
What to Look For in the Response
  • Does the agent handle a complex, multi-constraint scheduling request?
  • Does it prioritize HIGH tier HCPs as instructed?
  • Does it factor in geography (downtown core clustering)?
  • Does it produce a realistic, time-bound schedule?
  • Red flag: Can only handle simple HCP lookup queries
Prompt 5C — Predictive Capability
Paste this prompt into your agent
Based on current engagement trends, which 5 HCPs in my territory
are most likely to give consent if approached in the next 30 days?
What to Look For in the Response
  • Does the agent identify signals that predict consent likelihood?
  • Does it provide a specific list of 5 HCPs with reasoning?
  • Does it go beyond current scoring to forward-looking predictions?
  • Does it explain why these HCPs are likely to consent now?
  • Red flag: No predictive capability, only reactive lookup

Red Flags & Success Indicators

❌ Avoid These Red Flags

  • Agent ignores CASL context or gives generic consent advice
  • Scoring logic is a black box — agent can’t explain why an HCP is HIGH/MED/LOW
  • No differentiation between consent strategies per tier
  • Agent fails on missing data or edge cases (declined consent, new HCP)

✅ Must-Have Success Indicators

  • Agent ranks HCPs using both CRM and IQVIA data
  • Agent explains prioritization logic clearly and defensibly
  • Agent recommends differentiated strategies per tier
  • Agent handles real-world data gaps and edge cases intelligently
